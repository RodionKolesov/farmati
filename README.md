# Косметический бренд: свой сайт + личный кабинет с бонусами

Кастомный сайт-магазин (витрина, корзина, оплата) + личный кабинет с **бонусной
системой** на **Supabase** и отдельном SPA. Всё своё, без ограничений сторонних песочниц:
полный контроль над дизайном, данными и логикой.

```
Свой сайт (бренд.ru)        Supabase                     Кабинет (lk.бренд.ru)
 каталог + корзина  ──webhook──►  RPC/бэкенд  ──►  Postgres  ◄──API──  React SPA
 кнопка «Кабинет» ───────────────────────────────────────────────────►   вход/баланс/промокоды
```

## Структура репозитория

```
cosmetics-loyalty/
├─ web/                                      # витрина-магазин (Next.js): каталог, корзина, чекаут, курсы, кабинет
├─ supabase/
│  └─ migrations/0001_init.sql              # таблицы, RLS, триггеры, RPC бонусной системы
├─ cabinet/                                 # SPA личного кабинета (React + Vite)
│  ├─ src/{App.jsx, supabaseClient.js, ...}
│  └─ .env.example
└─ README.md
```

## Шаг 1. Supabase (бэкенд бонусов)

1. Создайте проект на https://supabase.com.
2. Примените схему: SQL Editor → вставьте содержимое `supabase/migrations/0001_init.sql` → Run.
   (или через CLI: `supabase link` → `supabase db push`).
3. Authentication → Providers → включите **Email**. По желанию включите подтверждение почты.
4. Project Settings → API — скопируйте `Project URL` и ключи `anon` / `service_role`.

### Приём заказов с сайта

Бонусы начисляет RPC `accrue_order_bonus`. Ваш сайт после оплаты заказа вызывает её
(с ключом `service_role` на стороне сервера — не из браузера!), передавая
`external_order_id`, email/телефон покупателя и сумму. Идемпотентность — по
`external_order_id` (повторный вызов не задвоит бонусы).

## Шаг 2. Кабинет (SPA)

```bash
cd cabinet
cp .env.example .env        # впишите VITE_SUPABASE_URL и VITE_SUPABASE_ANON_KEY
npm install
npm run dev                 # http://localhost:5173
npm run build               # прод-сборка в dist/ (залить на хостинг, домен lk.бренд.ru)
```

Что умеет: регистрация/вход, баланс бонусов, история заказов и движений бонусов,
кнопка «списать бонусы в промокод».

## Шаг 3. Витрина (свой сайт)

- **Каталог и корзина** — в приложении `web/` (Next.js): свои карточки товаров, корзина,
  чекаут и эквайринг.
- **Кнопка «Личный кабинет»** в шапке → ссылка на `https://lk.бренд.ru`.
- **После оплаты заказа** сайт вызывает RPC `accrue_order_bonus` (см. Шаг 1) — бонусы начисляются.

## Как работают бонусы

1. Клиент оформляет заказ на сайте → после оплаты сайт вызывает RPC `accrue_order_bonus`.
2. Функция находит пользователя по **email** (или по последним 10 цифрам телефона),
   создаёт заказ и начисляет `earn_rate` от суммы.
   Идемпотентность — по `external_order_id` (повторный вызов не задвоит бонусы).
3. В кабинете клиент видит баланс и **списывает бонусы в промокод** (RPC
   `redeem_bonus_to_promo`): баланс уменьшается, создаётся код вида `BNS-XXXXXXXX`.
4. Этот код применяется как скидочный промокод в корзине сайта.

> Сопоставление «заказ ↔ пользователь» идёт по email/телефону — просите клиента
> указывать при заказе те же контакты, что и при регистрации в кабинете.

## Проверка end-to-end

1. Зарегистрируйтесь в кабинете (`npm run dev`) — в таблице `profiles` появится строка.
2. Сэмулируйте заказ (тем же email) — вызовите RPC из SQL Editor Supabase:
   ```sql
   select * from public.accrue_order_bonus(
     p_external_order_id => 'test-1',
     p_email             => 'you@mail.ru',
     p_phone             => '+79990000000',
     p_amount            => 2000
   );
   ```
3. Обновите кабинет → баланс +100 (5% от 2000), заказ и транзакция в истории.
4. Спишите 50 в промокод → баланс 50, появился код `BNS-…`.
5. Повторите тот же вызов с `p_external_order_id => 'test-1'` → бонусы НЕ задвоятся (идемпотентность).
