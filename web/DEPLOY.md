# Деплой FARMATI — пошагово

Сайт на Next.js. Локально работает на SQLite. Для хостинга нужно: база Postgres,
переменные окружения, сборка и размещение. Ниже два пути: **VPS** (рекомендуется —
дешевле, и загрузка фото работает) и **Vercel** (проще, но фото нужно в облако).

---

## 1. База данных → Postgres

1. Заведите Postgres: **Neon** (neon.tech, бесплатный тариф) или на своём VPS.
2. В `prisma/schema.prisma` поменяйте провайдер:
   ```prisma
   datasource db {
     provider = "postgresql"   // было: sqlite
     url      = env("DATABASE_URL")
   }
   ```
3. В `.env` укажите строку подключения:
   ```
   DATABASE_URL="postgresql://user:pass@host/db?sslmode=require"
   ```
4. Примените схему и наполните данными:
   ```bash
   npx prisma db push
   node prisma/seed.mjs        # стартовые товары/курсы (по желанию)
   node scripts/make-admin.mjs # создать админа
   ```

## 2. Переменные окружения (.env)

```
DATABASE_URL=...                      # Postgres
AUTH_SECRET=...                       # сгенерировать: npx auth secret
AUTH_TRUST_HOST=true
NEXT_PUBLIC_BASE_URL=https://ваш-домен.ru
ADMIN_EMAIL=почта-владельца@mail.ru   # этот аккаунт видит /admin
YOOKASSA_SHOP_ID=...                  # из кабинета ЮKassa (когда подключите)
YOOKASSA_SECRET_KEY=...
```

## 3. Оплата (ЮKassa)

1. Зарегистрируйте магазин в ЮKassa (самозанятый/ИП/ООО), пройдите модерацию.
2. Скопируйте `shopId` и секретный ключ в `.env` (см. выше).
3. В настройках ЮKassa укажите URL вебхука:
   `https://ваш-домен.ru/api/payments/webhook`
4. Без ключей сайт работает в dev-режиме (заказ оформляется без реальной оплаты).

## 4а. Размещение на VPS (рекомендуется)

```bash
npm install
npm run build
npm run start            # поднимет на :3000
```
- Поставьте перед приложением **Nginx** (проксирование на 3000) и SSL (Let's Encrypt).
- Держите процесс живым через **pm2**: `pm2 start npm --name farmati -- start`.
- Загрузка фото товаров/курсов сохраняется в `public/products` — на VPS работает.

## 4б. Размещение на Vercel (проще)

1. Залейте проект в GitHub, импортируйте в Vercel.
2. Добавьте переменные окружения из п.2 в настройках проекта.
3. Vercel соберёт и задеплоит автоматически.
4. ⚠️ На Vercel файловая система временная — **загруженные фото не сохранятся**.
   Нужно облако для картинок (Cloudinary/S3) — подключается отдельно. Видео это не касается.

## 5. Домен

Привяжите домен к хостингу (VPS — A-запись на IP; Vercel — в настройках доменов).
Пропишите его в `NEXT_PUBLIC_BASE_URL`.

## 6. Что проверить после запуска

- [ ] Регистрация участника (ФИО + телефон + email)
- [ ] Добавление товара в корзину и оформление заказа с доставкой
- [ ] Списание бонусов в корзине, начисление 5% после оплаты
- [ ] Вход в админку под ADMIN_EMAIL → /admin
- [ ] Покупка курса → открылись платные уроки
- [ ] Форма консультации → заявка в /admin/leads
- [ ] Реальная оплата (после подключения ключей ЮKassa)

## 7. Перед публикацией (контент и право)

- Заменить тестового админа, поставить реальный ADMIN_EMAIL и сменить пароль.
- Заполнить реквизиты в /privacy и /offer.
- Добавить фото 2 товаров (дневной крем, тонер), реальные отзывы, контакты в подвале.
- Залить видео на VK/YouTube/Kinescope и вставить ссылки в уроки.
